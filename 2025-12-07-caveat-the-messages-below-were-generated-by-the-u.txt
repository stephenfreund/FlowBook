     Implementation Plan: enforce_monotone_updates Kernel Option

     Overview

     Add enforce_monotone_updates mode to FerretKernel. When enabled, 
     validates that read-before-write variables aren't modified during
     execution. If violated, state is reverted and an error is reported.

     Design Decisions (confirmed)

     - Check scope: Read-before-write variables only (from existing tracking
      data)
     - Enable method: Magic commands only
     - Execution count: Increment on failure

     Implementation (single file change)

     File: data_ferret/kernel/ferret_kernel.py

     1. Add kernel flag (line ~54)

     self._enforce_monotone_updates = False

     2. Add magic commands (after line ~128)

     @line_cell_magic
     def enable_monotone_updates(self, line: str, cell: str = "") -> None:
         """Enable monotone updates enforcement."""
         if not self._use_global_tracking:
             # Auto-enable tracking (required for RBW detection)
             self.enable_global_tracking("", "")
         self._enforce_monotone_updates = True
         self.display_icon_and_text("✅", "Monotone updates enforcement 
     enabled")

     @line_cell_magic
     def disable_monotone_updates(self, line: str, cell: str = "") -> None:
         """Disable monotone updates enforcement."""
         self._enforce_monotone_updates = False
         self.display_icon_and_text("✅", "Monotone updates enforcement 
     disabled")

     3. Modify do_execute() (around lines 310-343)

     Before execution (after line 313):
     if self._enforce_monotone_updates:
         self._checkpoint.save("_monotone_pre", self.shell.user_ns)

     After successful execution (before line 343 return result):
     if self._enforce_monotone_updates and result.get('status') != 'error':
         rbw_vars = set(tracking.get('reads_before_writes', [])) if tracking
      else set()
         if rbw_vars:
             pre = self._checkpoint.get("_monotone_pre")
             post = Checkpoint("_monotone_post", self.shell.user_ns, {})
             diff_result = Checkpoint.diff(pre, post, 
     keys_to_include=rbw_vars, use_leq=True)
             if diff_result.differences:
                 # Revert state
                 self._checkpoint.restore("_monotone_pre", 
     self.shell.user_ns)
                 # Format error
                 error_msg = f"Monotonicity violation: 
     {list(diff_result.differences.keys())}"
                 self.send_response(self.iopub_socket, 'error', {
                     'ename': 'MonotonicityError',
                     'evalue': error_msg,
                     'traceback': [error_msg]
                 })
                 result = {
                     'status': 'error',
                     'execution_count': self.execution_count,
                     'ename': 'MonotonicityError',
                     'evalue': error_msg,
                     'traceback': [error_msg]
                 }
         self._checkpoint.delete("_monotone_pre")

⏺ The issue is clear. The tracking captures:
  - reads_before_writes: ALL variables read before being written (includes
  read-only vars)
  - writes: ALL variables written

  For monotonicity, we only care about variables that were modified (read
  THEN written). Variables that were only read can't have been changed, so
  they don't need checking.

  Fix: Use reads_before_writes ∩ writes instead of just reads_before_writes.

     Fix: Always reset tracking at start of do_execute

     Problem

     RBW set contains stale variables from previous cells because tracking 
     is only reset at the END of cell execution (in _capture_tracking_data),
      and only when _use_global_tracking is True. Stale data accumulates if 
     tracking was disabled.

     Solution

     Always reset tracking at the START of do_execute, before any cell code 
     runs. This ensures each cell starts with clean tracking regardless of
     previous state.

     Implementation

     File: data_ferret/kernel/ferret_kernel.py

     1. Add helper method

     def _reset_tracking(self) -> None:
         """Reset tracking data if user_ns is a TrackingDict."""
         if isinstance(self.shell.user_ns, TrackingDict):
             self.shell.user_ns.reset_tracking()

     2. Call at start of do_execute

     In do_execute, after display_cell_id() and before any checkpoint saves:

     async def do_execute(...):
         self._extract_cell_id(cell_id, cell_meta)
         code, timeout = self._parse_timeout_from_code(code)
         self.display_cell_id()

         # Always reset tracking at start of cell execution
         self._reset_tracking()

         force_checkpoints = self._force_checkpoints
         # ... rest of method

     3. Remove redundant reset from enable_monotone_updates

     The enable_monotone_updates magic no longer needs to reset tracking 
     since it happens automatically at cell start.
