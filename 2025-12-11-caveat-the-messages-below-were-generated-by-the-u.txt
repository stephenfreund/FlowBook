     Plan: Add Duration Access to TimedOutputContext

     Objective

     Enhance TimedOutputContext to allow accessing the elapsed duration 
     after the context manager completes, while maintaining full backwards
     compatibility with existing usage patterns.

     User Requirements

     1. Access duration from context variable: Enable with timer(...) as t: 
     pattern
     2. Property or method access: Support t.duration() or ideally just t 
     (via __str__)
     3. Backwards compatibility: Don't break the 47+ existing usages in the 
     codebase

     Current Implementation Analysis

     File: data_ferret/util/output.py

     Existing TimedOutputContext Behavior (Lines 166-210)

     - __enter__ currently returns nothing (implicit None)
     - Duration calculated in __exit__ as (end_time - self.start_time) * 
     1000 milliseconds
     - Duration stored in self.outer.timings list when key is provided
     - Duration displayed in console as "X ms" when message is provided

     Current Usage Patterns

     1. Standard pattern (most common - 40+ usages):
     with timer(key="operation", message="Doing work"):
         do_work()
       - No variable capture, duration auto-logged
     2. Duration access pattern (server commands - ~10 usages):
     with self.timing_context() as get_elapsed:
         do_work()
     elapsed = get_elapsed() * 1000  # Returns seconds, multiply for 
     milliseconds
       - Uses NotebookCommand.timing_context() static method (different 
     implementation)

     Proposed Solution

     Return self + duration() method

     Changes to TimedOutputContext (lines 166-210):

     1. Modify __enter__ to return self:
     def __enter__(self):
         # ... existing code ...
         self.start_time = time.time()
         self._duration_ms = None  # Initialize as private attribute
         return self  # NEW: return self instead of implicit None
     2. Store duration in __exit__:
     def __exit__(self, exc_type, exc_value, traceback):
         with self.outer.lock:
             end_time = time.time()
             duration = (end_time - self.start_time) * 1000
             self._duration_ms = duration  # NEW: store for later access
             # ... rest of existing code unchanged ...
     3. Add duration() method:
     def duration(self) -> float:
         """
         Return elapsed duration in milliseconds.

         Returns:
             Duration in milliseconds as a float.

         Raises:
             ValueError: If called before the context has exited.
         """
         if self._duration_ms is None:
             raise ValueError("Duration not available - context not yet 
     exited")
         return self._duration_ms

     Usage Examples (New Capabilities)

     # Access duration via method (user's requested pattern)
     with timer(key="op", message="Operation") as t:
         do_work()
     print("duration", t.duration())  # "duration 123.456"

     # Use in expressions
     with timer(key="op", message="Operation") as t:
         do_work()
     if t.duration() > 1000:
         print("Too slow!")

     # Store for later reference
     with timer(key="op", message="Operation") as t:
         do_work()
         checkpoint = take_checkpoint()
     print(f"Checkpoint took {t.duration()} ms")

     Backwards Compatibility

     ✅ Fully backwards compatible:
     - Existing with timer(...): pattern continues to work (return value 
     ignored)
     - All 47+ existing usages work unchanged
     - self.timing_context() in NotebookCommand is a separate implementation
      (not affected)
     - Global output.timings collection unchanged
     - Quiet mode suppression unchanged

     Implementation Steps

     1. Edit data_ferret/util/output.py (TimedOutputContext class, lines 
     166-210):
       - Initialize self._duration_ms = None in __enter__
       - Return self from __enter__
       - Store self._duration_ms = duration in __exit__
       - Add duration() method
     2. Test the enhancement:
       - Verify existing code still works (run existing tests if available)
       - Test new usage patterns manually

     Critical Files

     - /Users/freund/other/DataFerret/data_ferret/util/output.py - Lines 
     166-210 (TimedOutputContext class)

     Design Decisions

     Why milliseconds for duration()?

     - Consistent with existing display format ("X ms")
     - Matches internal storage in self.outer.timings list
     - User expects milliseconds based on existing timer output

     Why private attribute _duration_ms?

     - Indicates internal state - users should call duration() method
     - Prevents accidental direct access before context exits
     - Follows Python convention for internal attributes

     Why ValueError if called before exit?

     - Clear error message for programming mistakes
     - Prevents returning misleading partial/zero durations
     - Makes the API contract explicit
