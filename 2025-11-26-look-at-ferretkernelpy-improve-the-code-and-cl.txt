     Current Issues

     1. Disorganized and Duplicate Imports

     - Imports are scattered across the top of the file in multiple blocks

     - Should be in a separate module for process management

     3. Mixed Responsibilities in FerretKernel Class

     The class handles too many concerns:
     - Display utilities: display_cell_id, display_icon_and_text, 
     diff_checkpoints
     - Cell magics: enable_scalene, disable_scalene, force_checkpoints, 
     checkpoint
     - Comm handlers: _debug_command_comm_open, _kernel_command_comm_open
     - Exception handling: _custom_exc_handler
     - Core execution: do_execute, do_scalene
     - AST manipulation: wrap_last_expr_with_print_repr
     - Text processing: replace_filenames_with_cell_ids, display_contents
     - JSON serialization: _make_json_safe

     4. Verbose Checkpoint Magic

     The checkpoint magic method is ~85 lines of repetitive argument parsing
      and handler calls.

     5. Complex do_execute Method

     The main execution method is ~130 lines with nested logic for:
     - Cell ID extraction
     - Timeout parsing
     - Checkpoint handling
     - Scalene profiling
     - Cleanup in finally block

     - DisplayHelper class with:

     - ScaleneRunner class with:

       - wrap_last_expr_with_print_repr(src) - AST transformation
       - replace_filenames_with_cell_ids(text, cell_id_map)
       - display_contents(filename)

     Step 5: Extract JSON Serialization Utility

     Create data_ferret/kernel/json_utils.py:
     - Move _make_json_safe() function
     - This handles numpy arrays, NaN values, infinity, etc.

     Step 6: Simplify Checkpoint Magic

     Refactor the checkpoint magic to use a dispatcher pattern:
     @line_cell_magic
     def checkpoint(self, line: str, cell: str = "") -> None:
         args = line.split()
         if not args:
             self._display.error("Usage: checkpoint <command> [args]")
             return

         cmd_handlers = {
             'save': self._checkpoint_save,
             'restore': self._checkpoint_restore,
             'delete': self._checkpoint_delete,
             'list': self._checkpoint_list,
             'compare': self._checkpoint_compare,
             'clear': self._checkpoint_clear,
         }

         handler = cmd_handlers.get(args[0])
         if handler:
             handler(args[1:])
         else:
             self._display.error(f"Unknown command: {args[0]}")

     Step 7: Simplify do_execute Method

     Extract helper methods:
     - _parse_cell_id(cell_meta, cell_id) - Extract cell ID from various 
     sources
     - _parse_timeout(code) - Parse and strip timeout comment from code
     - _create_execution_metadata(duration, profile_contents, type_models) -
      Build metadata dict
     - _handle_checkpoint_if_enabled() - Checkpoint save/diff logic

     Step 8: Clean Up FerretKernel Class

     After extractions, the class should:
     - Initialize dependencies in __init__
     - Have clear delegation to helper classes
     - Keep only the essential IPythonKernel overrides

     ├── kernel_commands.py     # Existing: request/response models
     ├── checkpoint.py          # Existing: checkpoint logic
     └── ... (other existing files)

     Implementation Order

     1. Step 1: Organize imports (low risk, immediate cleanup)
     2. Step 2: Extract process_cleanup.py (isolated, no dependencies)
     3. Step 5: Extract json_utils.py (small, standalone)
     4. Step 3: Extract display_helpers.py (used throughout)
     5. Step 4: Extract scalene_runner.py (complex but isolated)
     6. Step 6: Simplify checkpoint magic (depends on display helpers)
