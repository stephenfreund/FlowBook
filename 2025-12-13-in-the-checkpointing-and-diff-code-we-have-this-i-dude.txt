     Structural Warnings Implementation Plan

     Problem

     Structural warnings (generated in WARN mode when structural reads like 
     df.columns are followed by structural changes like adding columns) are:
     1. Generated by Checkpoint.diff() and stored in DiffResult.warnings
     2. But never extracted, logged, displayed to users, or included in 
     metadata

     Goal

     Make structural warnings visible throughout the system:
     1. Logged with error() for console visibility
     2. Displayed to user in notebook (like violations)
     3. Included in metadata returned to frontend
     4. Shown in metadata panel UI

  The issue was that _check_read_write_conflict had a gap in its logic:

  - Before: When both cells had column-level info, it only checked column data overlap. If no overlap, no conflict - even in ENFORCE mode with structural reads.
