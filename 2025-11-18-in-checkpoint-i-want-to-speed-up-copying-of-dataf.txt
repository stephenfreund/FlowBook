  Common Immutable Types

  Basic Python immutables:

  - range

  Numeric types:
  - All NumPy scalar types: np.int64, np.float64, np.bool_, etc.
  - decimal.Decimal

  Date/time types:
  - datetime.date, datetime.datetime, datetime.time, datetime.timedelta

  Other pandas types:
  - pd.NA, pd.NaT

  Implementation Strategy

  Here's an optimized approach:

  def is_immutable_type(obj) -> bool:
      """Check if an object is of an immutable type that's safe to skip copying."""
      if obj is None or obj is pd.NA:
          return True

      # Basic immutable types
      if isinstance(obj, (int, float, str, bool, bytes, complex, frozenset, range)):
          return True

      # NumPy scalar types (all are immutable)
      if isinstance(obj, np.generic):
          return True

      # Date/time types
      if isinstance(obj, (datetime.date, datetime.time, datetime.timedelta,
                         pd.Timestamp, pd.Timedelta, pd.Period)):
          return True

      # Decimal
      if isinstance(obj, decimal.Decimal):
          return True

      return False

  def is_column_all_immutable(series: pd.Series) -> bool:
      """Check if all non-null values in a Series are immutable types."""
      if len(series) == 0:
          return True

      # Sample a few values for quick check
      # If first value is immutable, check if all are same type
      non_null = series.dropna()
      if len(non_null) == 0:
          return True

      first_val = non_null.iloc[0]
      if not is_immutable_type(first_val):
          return False

      # Check if all values are same type (fast path)
      first_type = type(first_val)
      if all(type(x) is first_type for x in non_null.iloc[1:]):
          return True

      # Fallback: check each value individually
      return all(is_immutable_type(x) for x in non_null.iloc[1:])

  Immutable Copy Optimization for DataFrames

  I've added an optimization that speeds up checkpoint copying by skipping deep copy operations for immutable objects in DataFrame/Series object-dtype columns.

  Key Changes

  1. New Helper Functions (checkpoint.py:90-162):
  - is_immutable_type(obj): Checks if an object is immutable
  - is_column_all_immutable(series): Checks if all values in a column are immutable

  2. Immutable Types Recognized:
  - Basic Python: int, float, str, bool, None, bytes, complex, frozenset, range
  - NumPy: All scalar types (np.int64, np.float64, etc.)
  - Date/time: datetime.date, datetime.time, datetime.timedelta
  - Pandas: pd.Timestamp, pd.Timedelta, pd.Period, pd.NA
  - decimal.Decimal

  3. Flag-Based Control:
  - Checkpoints(skip_immutable_copy=True/False): Controls optimization
  - Default is False (conservative, backward-compatible)
  - Kernel uses True via self._skip_immutable_copy flag (ferret_kernel.py:363)

  4. Modified Copying Logic:
  - DataFrame object columns: Skip deepcopy if all values are immutable
  - Series object dtype: Skip deepcopy if all values are immutable
  - Mutable objects still get proper deep copying

  To Toggle the Optimization

  Simply change the flag in ferret_kernel.py:363:
  self._skip_immutable_copy = True   # Optimization enabled (default)
  self._skip_immutable_copy = False  # Optimization disabled

  Performance Impact

  For DataFrames with string/numeric/datetime columns (very common in data science), this avoids expensive deepcopy() calls on potentially millions of cells, significantly speeding up checkpoint operations.
