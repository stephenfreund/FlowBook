  IPython Kernel History Stack - Stored Information

  The IPython kernel's history system stores execution information in an
  SQLite database (history.sqlite) with the following structure:

  For Each Executed Piece of Code (stored in the history table):

  1. session (integer) - The session identifier that this execution belongs
  to
  2. line (integer) - The line/execution number within that session
  (essentially the execution count)
  3. source (text) - The transformed/processed source code after IPython has
   applied its transformations (magic commands processed, etc.)
  4. source_raw (text) - The raw source code exactly as entered by the user,
   before any IPython transformations

  Optional Output Storage (stored in the output_history table):

  If db_log_output is enabled:
  - session (integer) - Session identifier
  - line (integer) - Line number
  - output (text) - Text representation of the execution result (only
  captures the return value from the last expression, not error messages)

  Additional Context from the FerretKernel:

  In your DataFerret kernel specifically, you also have:
  - _executed_cell_ids (ferret_kernel.py:144) - A Dict[int, str] mapping
  execution counts to cell IDs (4-character cell identifiers like "abcd")

  This is separate from IPython's history but provides a mapping between
  execution counts and your notebook's cell IDs, used primarily by the
  ScaleneRunner for profiling error reporting.

  ---
  Key Points:
  - The history stores both raw and processed source code
  - Output storage is optional and disabled by default
  - Error messages are not stored in output_history
  - The primary keys are composite: (session, line)

that ensures notebooks behave like well-formed dataflow programs. Here's my

---

## **Sequential Dataflow Consistency (SDC)**

### Informal Definition
A notebook satisfies Sequential Dataflow Consistency if its execution 
behavior is equivalent to a single top-to-bottom run, dependencies only flow
 forward (downward), and re-execution properly invalidates dependent cells.

### Formal Definition

Let cells be **C₁, C₂, ..., Cₙ** in document order.

For each cell Cᵢ, define:
- **Reads(Cᵢ)**: the set of state variables read by Cᵢ
- **Writes(Cᵢ)**: the set of state variables written by Cᵢ

A notebook satisfies **Sequential Dataflow Consistency** iff:

**1. Reproducibility Invariant**  
Any valid execution sequence produces a final state identical to executing 
C₁, C₂, ..., Cₙ exactly once in order.

**2. Staleness Propagation Rule**  
If Cᵢ is (re-)executed, then Cⱼ (where j > i) becomes *stale* if:
- Reads(Cⱼ) ∩ Writes(Cᵢ) ≠ ∅, or
- Cⱼ depends on any stale cell (transitively)

**3. No Backward Mutation Constraint**  
For all cells Cⱼ:
$$\text{Writes}(C_j) \cap \left( \bigcup_{i < j} \text{Reads}(C_i) \right) =
 \emptyset$$

In other words: a cell may not modify any state that a preceding cell reads.

---

### Intuition

Rule 3 is the key enforcement mechanism—it ensures the dependency graph is a
 DAG that respects cell order. Without it, re-running an early cell could 
silently depend on mutations from later cells, breaking reproducibility.

### Alternative Names
Depending on what resonates with your team:
- **Monotonic Dataflow Constraint**
- **Forward-Only Dependency Model**
- **Topological Execution Consistency**

(like cells that mutate objects aliased across cells)?

⏺ Implementing Sequential Dataflow Consistency (SDC)

  This is a fascinating constraint system. Let me explore implementation
  approaches with a focus on simplicity and clarity.

  Core Challenge

  SDC Rule 3 requires: after executing cell Cⱼ, the values of variables read
   by any preceding cell Cᵢ (i < j) must remain unchanged.

  Key insight: We need to track what each cell reads, then verify those
  values aren't mutated by later cells.

  ---
  Implementation Approaches

  Approach 1: History-Based Verification (Recommended)

  Idea: Maintain a history of what each cell read and the values at
  read-time. After each cell execution, check if any historical reads were
  invalidated.

  Data Structures:
    history: Map<cell_id, List<(var_name, value_hash)>>  # what each cell
  read

  On cell Cⱼ execution:
    1. Execute Cⱼ, capturing Reads(Cⱼ)
    2. For each cell Cᵢ where i < j:
         For each (var, old_hash) in history[Cᵢ]:
           if hash(current_state[var]) ≠ old_hash:
             VIOLATION: Cⱼ mutated state read by Cᵢ
    3. Store history[Cⱼ] = [(var, hash(val)) for var in Reads(Cⱼ)]

  Pros:
  - Conceptually simple
  - Precisely captures the semantic definition
  - Works with your existing TrackingDict infrastructure

  Cons:
  - O(cells × variables) verification per execution
  - Requires hashable/comparable values

  ---
  Approach 2: Pre/Post Snapshot Comparison (Simpler)

  Idea: Before executing Cⱼ, snapshot all variables read by preceding cells.
   After execution, compare.

  Data Structures:
    all_prior_reads: Map<var_name, Set<cell_id>>  # which cells read each
  var

  On cell Cⱼ execution:
    1. prior_read_vars = union of Reads(Cᵢ) for all i < j
    2. pre_snapshot = {v: hash(state[v]) for v in prior_read_vars}
    3. Execute Cⱼ
    4. For v in prior_read_vars:
         if hash(state[v]) ≠ pre_snapshot[v]:
           VIOLATION: Cⱼ mutated v, which was read by cells
  {all_prior_reads[v]}
    5. Update all_prior_reads with Reads(Cⱼ)

  Pros:
  - Single pass verification
  - Clear mental model: "did this cell change anything earlier cells depend
  on?"
  - Naturally integrates with checkpointing

  Cons:
  - Must snapshot before execution (can't be purely post-hoc)
  - Snapshot cost grows with notebook size

  ---
  Approach 3: Incremental Write-Set Intersection (Lightest)

  Idea: Track the union of all prior reads. Check if current writes
  intersect.

  Data Structures:
    prior_reads_union: Set<var_name>

  On cell Cⱼ execution:
    1. Execute Cⱼ, capturing Writes(Cⱼ)
    2. violation_vars = Writes(Cⱼ) ∩ prior_reads_union
    3. If violation_vars non-empty: VIOLATION
    4. prior_reads_union = prior_reads_union ∪ Reads(Cⱼ)

  Pros:
  - O(1) state, O(|writes|) check per cell
  - Trivial to implement
  - No snapshotting needed

  Cons:
  - Syntactic, not semantic — flags x = x as violation
  - Can't distinguish "wrote same value" from "wrote different value"

  ---
  Recommendation: Hybrid Approach

  Combine Approach 3's efficiency with Approach 2's precision:

  class SDCEnforcer:
      def __init__(self):
          self.prior_reads: Dict[str, Any] = {}  # var -> last-read value 
  hash
          self.var_to_cells: Dict[str, Set[str]] = {}  # var -> cells that 
  read it

      def pre_execute(self, cell_id: str, state: dict) -> dict:
          """Snapshot values of variables read by prior cells."""
          return {v: self._hash(state.get(v)) for v in self.prior_reads}

      def post_execute(self, cell_id: str, state: dict, 
                       pre_snapshot: dict, reads: Set[str]) -> 
  Optional[Violation]:
          """Check for backward mutations, update tracking."""
          # Check: did we mutate anything a prior cell read?
          for var, old_hash in pre_snapshot.items():
              new_hash = self._hash(state.get(var))
              if new_hash != old_hash:
                  return Violation(
                      cell=cell_id,
                      var=var,
                      read_by=self.var_to_cells[var]
                  )

          # Update: record what this cell reads
          for var in reads:
              self.prior_reads[var] = self._hash(state.get(var))
              self.var_to_cells.setdefault(var, set()).add(cell_id)

          return None

  ---
  Design Tradeoffs

  | Dimension       | Simple (Approach 3) | Precise (Approach 2) | Hybrid
        |
  |-----------------|---------------------|----------------------|----------
  ------|
  | False positives | Yes (x=x)           | No                   | No
        |
  | False negatives | No                  | No                   | No
        |
  | Memory          | O(vars)             | O(cells × vars)      | O(vars)
        |
  | Per-cell cost   | O(writes)           | O(prior_reads)       |
  O(prior_reads) |
  | Implementation  | ~20 lines           | ~50 lines            | ~40 lines
        |

  ---
  Key Implementation Questions

  1. What counts as "equality"?

  Options (increasing precision, increasing cost):
  - Identity: id(pre) == id(post) — too strict, copies break it
  - Hash: hash(pickle.dumps(obj)) — handles most cases
  - Deep equality: Your existing equality.py — most precise
  - Structural hash: Hash shape + sample values for DataFrames

  Recommendation: Use hash for primitives, structural hash for DataFrames
  (since deep equality is expensive for large data).

  2. How to handle aliasing?

  # Cell 1
  df = pd.DataFrame(...)
  summary = df  # alias!

  # Cell 2 reads summary
  print(summary.shape)

  # Cell 3 mutates df
  df['new_col'] = 1  # Also mutates summary!

  Options:
  - Track at object-id level, not variable-name level
  - Snapshot values, not references (more expensive but correct)
  - Document as limitation (simplest)

  Recommendation: Snapshot values. Your checkpoint system already does this.

  3. When to check?

  - Eager (during execution): Instrument __setitem__ on namespace
  - Lazy (after execution): Compare pre/post snapshots

  Recommendation: Lazy. Simpler, and you already have checkpointing
  infrastructure.

  4. How to handle cell reordering/deletion?

  When cells move or are deleted, prior_reads must be recomputed. Options:
  - Recompute from scratch on structural change
  - Maintain per-cell read sets, rebuild union incrementally

  Recommendation: Store per-cell read sets, rebuild union on change. Your
  _executed_cell_ids pattern already tracks execution order.

  Simplest Viable Implementation

  If you want maximum clarity with minimal code:

  class SDCEnforcer:
      """Enforces Sequential Dataflow Consistency Rule 3."""

      def __init__(self):
          self.prior_read_values: Dict[str, bytes] = {}  # var -> pickled 
  value

      def check_and_update(self, state: dict, reads: Set[str]) -> 
  Optional[str]:
          """
          Call after cell execution.
          Returns error message if violation, None otherwise.
          """
          # Check: did any prior-read variable change?
          for var, old_pickle in self.prior_read_values.items():
              if var in state:
                  new_pickle = pickle.dumps(state[var])
                  if new_pickle != old_pickle:
                      return f"SDC violation: {var} was modified"

          # Update: record current cell's reads
          for var in reads:
              if var in state:
                  self.prior_read_values[var] = pickle.dumps(state[var])

          return None

  3. Computing Staleness and Client Communication

  This is the core algorithm. Let me think through it carefully.

  The Problem

  When cell Cⱼ is (re-)executed:
  1. Some variables may have changed
  2. Any cell that previously read those variables is now stale
  3. Staleness propagates transitively

  Algorithm

  @dataclass
  class ExecutionRecord:
      cell_id: str
      reads: Set[str]                    # variables read by this cell
      pre_checkpoint: str                # checkpoint name before execution
      execution_seq: int                 # global sequence number when 
  executed

  @dataclass 
  class SDCState:
      records: Dict[str, ExecutionRecord]  # cell_id -> most recent 
  execution
      seq_counter: int = 0                  # monotonic execution counter
      checkpoints: Checkpoints             # reference to kernel's 
  checkpoint manager

  def on_cell_execute(self, cell_id: str, reads: Set[str], 
                      pre_checkpoint: str, post_state: dict) -> SDCResult:
      """
      Called after cell execution completes.
      Returns violations and stale cells.
      """
      self.seq_counter += 1
      old_record = self.records.get(cell_id)

      # 1. Check SDC Rule 3: did we mutate anything prior cells read?
      violation = self._check_backward_mutation(cell_id, pre_checkpoint,
  post_state)

      # 2. Compute which cells are now stale
      stale_cells = self._compute_stale_cells(cell_id, pre_checkpoint,
  post_state)

      # 3. Update our record for this cell
      self.records[cell_id] = ExecutionRecord(
          cell_id=cell_id,
          reads=reads,
          pre_checkpoint=pre_checkpoint,
          execution_seq=self.seq_counter,
      )

      return SDCResult(violation=violation, stale_cells=stale_cells)

  Computing Stale Cells

  def _compute_stale_cells(self, executed_cell: str, 
                           pre_checkpoint: str, post_state: dict) -> 
  List[str]:
      """
      Find cells that read variables which changed.
      Uses transitive closure for propagation.
      """
      pre_state = self.checkpoints.get(pre_checkpoint)

      # What variables changed?
      changed_vars = self._diff_changed_vars(pre_state, post_state)
      if not changed_vars:
          return []

      # Which cells (executed after this one previously) read those vars?
      my_prev_seq = self.records.get(executed_cell,
  ExecutionRecord(...)).execution_seq

      directly_stale = set()
      for cell_id, record in self.records.items():
          if cell_id == executed_cell:
              continue
          # Only cells that ran after our previous execution are candidates
          if record.execution_seq > my_prev_seq:
              if record.reads & changed_vars:  # intersection
                  directly_stale.add(cell_id)

      # Transitive closure: cells that read outputs of stale cells
      return self._propagate_staleness(directly_stale)

  def _propagate_staleness(self, initial_stale: Set[str]) -> List[str]:
      """Compute transitive closure of staleness."""
      stale = set(initial_stale)
      changed = True

      while changed:
          changed = False
          for cell_id, record in self.records.items():
              if cell_id in stale:
                  continue
              # If this cell reads any var written by a stale cell, it's 
  stale too
              for stale_cell in stale:
                  stale_writes = self._get_writes(stale_cell)
                  if record.reads & stale_writes:
                      stale.add(cell_id)
                      changed = True
                      break

      # Return in execution order for client
      return sorted(stale, key=lambda c: self.records[c].execution_seq)

  Checking Backward Mutations (Rule 3)

  def _check_backward_mutation(self, executed_cell: str,
                                pre_checkpoint: str, post_state: dict) -> 
  Optional[Violation]:
      """
      Check if this cell mutated variables read by cells executed before it
      (in document order, not execution order).
      """
      pre_state = self.checkpoints.get(pre_checkpoint)
      changed_vars = self._diff_changed_vars(pre_state, post_state)

      if not changed_vars:
          return None

      # Get document order (cell positions in notebook)
      my_position = self._get_cell_position(executed_cell)

      # Check cells that appear BEFORE us in document order
      for cell_id, record in self.records.items():
          cell_position = self._get_cell_position(cell_id)
          if cell_position < my_position:
              violated_vars = record.reads & changed_vars
              if violated_vars:
                  return Violation(
                      mutating_cell=executed_cell,
                      affected_cell=cell_id,
                      variables=violated_vars,
                      message=f"Cell {executed_cell} modified 
  {violated_vars} "
                              f"which cell {cell_id} (earlier in notebook) 
  reads"
                  )

      return None

  Communicating to Client

  @dataclass
  class SDCResult:
      violation: Optional[Violation]      # Rule 3 violation if any
      stale_cells: List[str]              # Cell IDs that need re-execution
      changed_variables: Set[str]         # What changed (for UI 
  highlighting)

  # In kernel, after execution:
  result = self.sdc.on_cell_execute(cell_id, tracking.reads_before_writes,

  self.shell.user_ns)

  # Send to client via iopub or display
  if result.stale_cells:

          'stale_cells': result.stale_cells,

          'reason': f'Cell {cell_id} modified shared state'
      })

          'variables': list(result.violation.variables),
      })

  Key Design Decisions

  Document Order vs Execution Order

  SDC Rule 3 uses document order (cell position in notebook), but staleness
  uses execution order (when cells were last run). These are different:

  Document order: [A, B, C, D]
  Execution order: A(1), C(2), B(3), D(4)  # user ran out of order

  If A is re-run:
  - Rule 3 check: Did A mutate vars read by cells BEFORE A? (none exist)
  - Staleness: Which cells ran AFTER A's previous run (seq=1) and read A's
  outputs?
              That's C, B, D — check each against changed vars

  What About Cell Position Changes?

  If user reorders cells in the notebook, document order changes. Options:

  1. Invalidate all SDC state — simple, conservative
  2. Recompute based on new positions — more complex but smoother UX
  3. Track by execution order only — ignore document order (weaker
  guarantee)

  Recommendation: Option 1 for now. Cell reordering is relatively rare and a
   clean slate is easier to reason about.

  Handling Writes Tracking

  The algorithm above assumes we can get writes for any cell. Options:

  1. Store writes in ExecutionRecord — requires tracking writes (you have
  this)
  2. Infer from checkpoints — diff pre/post to see what changed
  3. Conservative: all assigned vars — from AST analysis

  Recommendation: Store writes from TrackingDict. You already capture this.

  Simplified Integration

  class SDCEnforcer:

  infrastructure."""

  Callable):

          self.get_cell_position = get_cell_position  # cell_id -> int
          self.records: Dict[str, ExecutionRecord] = {}
          self.seq_counter = 0

      def pre_execute(self, cell_id: str, namespace: dict) -> str:
          """Call before execution. Returns checkpoint name."""
          checkpoint_name = f"_sdc_pre_{cell_id}"
          self.checkpoints.save(checkpoint_name, namespace)
          return checkpoint_name

      def post_execute(self, cell_id: str, namespace: dict,
                       pre_checkpoint: str, reads: Set[str], 
                       writes: Set[str]) -> SDCResult:
          """Call after execution. Returns violations and stale cells."""
          self.seq_counter += 1

          pre_state = self.checkpoints.get(pre_checkpoint)
          changed_vars = self._compute_changed(pre_state, namespace, writes)

          # Check Rule 3
          violation = self._check_rule3(cell_id, changed_vars)

          # Compute staleness  
          stale = self._compute_stale(cell_id, changed_vars)

          # Update record
          self.records[cell_id] = ExecutionRecord(
              cell_id=cell_id,
              reads=reads,
              writes=writes,
              pre_checkpoint=pre_checkpoint,
              execution_seq=self.seq_counter,
          )

          # Cleanup checkpoint
          self.checkpoints.delete(pre_checkpoint)

          return SDCResult(violation=violation, stale_cells=stale,
                          changed_variables=changed_vars)

      def _compute_changed(self, pre: Checkpoint, post_ns: dict, 
                           writes: Set[str]) -> Set[str]:
          """Use diff to find actually-changed variables."""
          changed = set()
          for var in writes:
              if var not in pre.data and var in post_ns:
                  changed.add(var)  # new variable
              elif var in pre.data and var not in post_ns:
                  changed.add(var)  # deleted
              elif var in pre.data and var in post_ns:
                  if not self._values_equal(pre.data[var], post_ns[var]):
                      changed.add(var)
          return changed

⏺ Cell Order Communication: Design Exploration

  The Problem

  The kernel needs to know document order (cell positions in notebook) to
  enforce SDC Rule 3. Currently, the kernel only sees cells as they execute
  — it has no knowledge of notebook structure.

  Option 1: Pass Cell Order in Execution Metadata

  Approach: Frontend sends cell_order: List[str] with each execution
  request.

  # In do_execute, cell_meta would include:
  cell_meta = {
      'cell_id': 'abcd',
      'cell_order': ['wxyz', 'abcd', 'efgh', 'ijkl'],  # full notebook order
      # ... other metadata
  }

  Pros:
  - Simple, explicit
  - Kernel always has current truth
  - No synchronization issues
  - Works immediately — no new protocol needed

  Cons:
  - Redundant data on every execution
  - Slight overhead (negligible for typical notebooks)

  Simplifications enabled:
  - get_cell_position(cell_id) becomes cell_order.index(cell_id)
  - No need to track/infer positions
  - Cell reordering automatically handled — next execution has new order

  ---
  Option 2: Notify Kernel of Structure Changes

  Approach: Frontend sends a message when notebook structure changes (cell
  add/delete/reorder).

  # New comm or message type
  {
      'type': 'notebook_structure',
      'cell_order': ['wxyz', 'abcd', 'efgh', 'ijkl'],
      'change': 'reorder'  # or 'add', 'delete'
  }

  Pros:
  - Only sent when structure changes (less frequent)
  - Kernel can react to changes proactively

  Cons:
  - Must ensure synchronization (what if message lost/delayed?)
  - More complex protocol
  - Edge cases: what if execution arrives before structure update?

  ---
  Option 3: Kernel Requests Order When Needed

  Approach: Kernel asks frontend for current order via comm.

  # Kernel sends request
  comm.send({'type': 'request_cell_order'})

  # Frontend responds
  comm.send({'type': 'cell_order', 'order': ['wxyz', 'abcd', ...]})

  Pros:
  - Kernel only asks when it cares
  - Frontend is source of truth

  Cons:
  - Async complexity — must wait for response
  - Adds latency to execution
  - Comm round-trip in hot path

  ---
  Option 4: Infer from Execution History

  Approach: Kernel infers order from the sequence of executions, assuming
  users generally execute top-to-bottom.

  Pros:
  - No protocol changes

  Cons:
  - Wrong when users execute out of order (common!)
  - Wrong when cells are reordered
  - Fundamentally unreliable

  ---
  Recommendation: Option 1 (Pass in Metadata)

  The overhead is negligible and the simplicity is substantial. Let me
  explore how this simplifies the design.

  ---
  Simplified SDC Design with Cell Order in Metadata

  Core Data Structures

  @dataclass
  class ExecutionRecord:
      cell_id: str
      reads: Set[str]
      writes: Set[str]
      execution_seq: int          # when this cell was last run

  @dataclass
  class SDCResult:
      violation: Optional[Violation]
      stale_cells: List[str]      # in document order
      changed_variables: Set[str]

  The Enforcer (Simplified)

  class SDCEnforcer:
      def __init__(self, checkpoints: Checkpoints):
          self.checkpoints = checkpoints
          self.records: Dict[str, ExecutionRecord] = {}
          self.seq_counter = 0

      def check(
          self,
          cell_id: str,
          cell_order: List[str],      # <-- passed from frontend
          pre_checkpoint: Checkpoint,
          post_namespace: dict,
          reads: Set[str],
          writes: Set[str],
      ) -> SDCResult:
          """
          Main entry point. Call after cell execution.
          """
          self.seq_counter += 1
          my_position = cell_order.index(cell_id)

          # Compute what actually changed
          changed = self._diff_changed(pre_checkpoint, post_namespace,
  writes)

          # Rule 3: Check backward mutation
          violation = self._check_backward_mutation(
              cell_id, my_position, cell_order, changed
          )

          # Staleness: Find affected downstream cells
          stale = self._compute_stale(cell_id, cell_order, changed)

          # Update record
          self.records[cell_id] = ExecutionRecord(
              cell_id=cell_id,
              reads=reads,
              writes=writes,
              execution_seq=self.seq_counter,
          )

          return SDCResult(
              violation=violation,
              stale_cells=stale,
              changed_variables=changed,
          )

  Rule 3 Check (Clean)

  def _check_backward_mutation(
      self,
      cell_id: str,
      my_position: int,
      cell_order: List[str],
      changed: Set[str],
  ) -> Optional[Violation]:
      """
      Did this cell mutate variables read by preceding cells?
      'Preceding' means earlier in document order (cell_order).
      """
      if not changed:
          return None

      # Check cells before us in document order
      for prior_cell_id in cell_order[:my_position]:
          record = self.records.get(prior_cell_id)
          if record is None:
              continue  # cell hasn't been executed yet

          violated = record.reads & changed
          if violated:
              return Violation(
                  mutating_cell=cell_id,
                  affected_cell=prior_cell_id,
                  variables=violated,
              )

      return None

  Staleness Computation (Clean)

  def _compute_stale(
      self,
      cell_id: str,
      cell_order: List[str],
      changed: Set[str],
  ) -> List[str]:
      """
      Find cells that need re-execution because their inputs changed.
      Returns cell IDs in document order.
      """
      if not changed:
          return []

      my_prev_seq = self._get_previous_seq(cell_id)

      # Direct staleness: cells that ran after us and read changed vars
      directly_stale = set()
      for other_id, record in self.records.items():
          if other_id == cell_id:
              continue
          if record.execution_seq > my_prev_seq:
              if record.reads & changed:
                  directly_stale.add(other_id)

      # Transitive staleness
      all_stale = self._propagate_staleness(directly_stale)

      # Return in document order (using cell_order)
      return [c for c in cell_order if c in all_stale]

  def _propagate_staleness(self, initial: Set[str]) -> Set[str]:
      """Transitive closure: if A is stale and B reads A's outputs, B is 
  stale."""
      stale = set(initial)

      changed = True
      while changed:
          changed = False
          for cell_id, record in self.records.items():
              if cell_id in stale:
                  continue
              # Does this cell read outputs of any stale cell?
              for stale_id in list(stale):
                  stale_writes = self.records[stale_id].writes
                  if record.reads & stale_writes:
                      stale.add(cell_id)
                      changed = True
                      break

      return stale

  ---
  What About Cells Not Yet Executed?

  An interesting case: cell exists in cell_order but has no ExecutionRecord.

  Scenarios:
  1. New cell added — not in records, should be ignored for Rule 3
  2. Cell never run — same, no reads/writes to check
  3. Cell was run, then deleted, then re-added — old record may be stale

  Design decision: Only cells in records participate in SDC checks.
  Unexecuted cells are neither stale nor capable of being violated.

  # In _check_backward_mutation:
  record = self.records.get(prior_cell_id)
  if record is None:
      continue  # not executed, skip

  ---
  Handling Cell Deletion

  When a cell is deleted from the notebook, it disappears from cell_order.
  Options:

  Option A: Lazy cleanup
  # On each check, prune records for cells no longer in notebook
  def _prune_deleted(self, cell_order: List[str]):
      current_cells = set(cell_order)
      deleted = [c for c in self.records if c not in current_cells]
      for c in deleted:
          del self.records[c]

  Option B: Keep records, just exclude from checks
  # Records persist, but only cells in current cell_order are considered
  # Pro: if cell is re-added, history preserved
  # Con: memory grows unboundedly (minor concern)

  Recommendation: Option A (lazy cleanup). Simpler mental model — records
  mirror executed cells in the current notebook.

  ---
  Handling Cell Reordering

  With cell_order passed on each execution, reordering is automatic:

  Before reorder: cell_order = [A, B, C]
  User moves C before B
  After reorder: cell_order = [A, C, B]

  Next execution of any cell sees new order.
  No explicit notification needed.

  But wait: What if reordering creates a new Rule 3 violation?

  Example:
  Before: [A, B] where A reads x, B writes x → valid (B after A)
  After:  [B, A] where A reads x, B writes x → NOW invalid (B before A)

  Question: Should reordering alone trigger violation detection?

  Options:
  1. Only check on execution — violations surface when B runs next
  2. Check on structure change — requires Option 2 (notify kernel of
  changes)
  3. Client-side pre-check — frontend validates before allowing reorder

  Recommendation: Option 1 for simplicity. The violation will be caught on
  next execution of B. This is consistent with the overall model: SDC is
  enforced at execution time.

  ---
  Alternative: Minimal Cell Order (Just Position)

  Instead of full cell_order, pass just the executing cell's position:

  cell_meta = {
      'cell_id': 'abcd',
      'cell_position': 2,      # 0-indexed position in notebook
      'total_cells': 5,
  }

  Pros:
  - Even less data per message

  Cons:
  - Can't return stale cells in document order (don't know other cells'
  positions)
  - Can't prune deleted cells (don't know which cells exist)
  - Loses ability to iterate "cells before me"

  Verdict: Not sufficient. Full cell_order is needed for the staleness
  return value and deletion handling.

  ---
  Alternative: Cell Order + Hashes for Change Detection

  Could extend cell_order with content hashes to detect code changes:

  cell_meta = {
      'cell_id': 'abcd',
      'cell_order': ['wxyz', 'abcd', 'efgh'],
      'cell_hashes': {'wxyz': 'a1b2', 'abcd': 'c3d4', 'efgh': 'e5f6'},
  }

  Use case: If cell code changed but wasn't re-executed, its record may be

  def notebook_structure(self, line: str, cell: str = "") -> None:

  """
  SDC Enforcer - Sequential Dataflow Consistency enforcement.

  Implements the three SDC rules:
  1. Reproducibility Invariant (structural)
  2. Staleness Propagation Rule (computed here)
  3. No Backward Mutation Constraint (enforced here)
  """

  from dataclasses import dataclass, field
  from typing import Dict, List, Optional, Set

  from data_ferret.kernel.checkpoint import Checkpoint, Checkpoints
  from data_ferret.kernel.models import SDCExecutionRecord, SDCResult,
  SDCViolation


  class SDCEnforcer:
      """
      Enforces Sequential Dataflow Consistency.
      
      Tracks cell executions and their read/write sets.
      On each execution, checks for backward mutations and computes 
  staleness.
      """

      def __init__(self, checkpoints: Checkpoints):
          self.checkpoints = checkpoints
          self.records: Dict[str, SDCExecutionRecord] = {}
          self.seq_counter: int = 0
          self._cell_order: List[str] = []

      @property
      def cell_order(self) -> List[str]:
          return self._cell_order

      def set_cell_order(self, order: List[str]) -> None:
          """Update notebook structure. Called via magic or metadata."""
          self._cell_order = order
          self._prune_deleted_cells()

      def _prune_deleted_cells(self) -> None:
          """Remove records for cells no longer in notebook."""
          current = set(self._cell_order)
          deleted = [c for c in self.records if c not in current]
          for c in deleted:
              del self.records[c]

      def check(
          self,
          cell_id: str,
          pre_checkpoint: Checkpoint,
          post_namespace: dict,
          reads: Set[str],
          writes: Set[str],
      ) -> SDCResult:
          """
          Main entry point. Call after cell execution.
          
          Args:
              cell_id: ID of the cell that just executed
              pre_checkpoint: Snapshot of namespace before execution
              post_namespace: Current namespace after execution
              reads: Variables read by the cell (reads_before_writes)
              writes: Variables written by the cell
              
          Returns:
              SDCResult with violation info and stale cells
          """
          self.seq_counter += 1

          # Compute what actually changed (semantic, not syntactic)
          changed = self._compute_changed(pre_checkpoint, post_namespace,
  writes)

          # Get position in document order
          try:
              my_position = self._cell_order.index(cell_id)
          except ValueError:
              # Cell not in order list — can't enforce SDC
              my_position = -1

          # Rule 3: Check backward mutation
          violation = None
          if my_position >= 0:
              violation = self._check_backward_mutation(cell_id,
  my_position, changed)

          # Compute staleness (cells that need re-execution)
          stale = self._compute_stale(cell_id, changed)

          # Update record for this cell
          self.records[cell_id] = SDCExecutionRecord(
              cell_id=cell_id,
              reads=reads,
              writes=writes,
              execution_seq=self.seq_counter,
          )

          return SDCResult(
              violation=violation,
              stale_cells=stale,
              changed_variables=sorted(changed),
          )

      def _compute_changed(
          self,
          pre: Checkpoint,
          post_ns: dict,
          writes: Set[str],
      ) -> Set[str]:
          """
          Compute variables whose values actually changed.
          Uses checkpoint diff for semantic comparison.
          """
          changed = set()

          for var in writes:
              pre_val = pre.data.get(var)
              post_val = post_ns.get(var)

              # New variable
              if var not in pre.data and var in post_ns:
                  changed.add(var)
              # Deleted variable
              elif var in pre.data and var not in post_ns:
                  changed.add(var)
              # Possibly modified
              elif var in pre.data and var in post_ns:
                  if not self._values_equal(pre_val, post_val):
                      changed.add(var)

          return changed

      def _values_equal(self, a, b) -> bool:
          """
          Check if two values are equal for SDC purposes.
          Uses checkpoint's equality logic.
          """
          from data_ferret.kernel.equality import equal
          try:
              return equal(a, b)
          except Exception:
              # If comparison fails, assume different
              return False

      def _check_backward_mutation(
          self,
          cell_id: str,
          my_position: int,
          changed: Set[str],
      ) -> Optional[SDCViolation]:
          """
          Rule 3: Did this cell mutate variables read by preceding cells?
          """
          if not changed:
              return None

          # Check cells before us in document order
          for prior_cell_id in self._cell_order[:my_position]:
              record = self.records.get(prior_cell_id)
              if record is None:
                  continue  # Not executed yet

              violated = record.reads & changed
              if violated:
                  return SDCViolation(
                      mutating_cell=cell_id,
                      affected_cell=prior_cell_id,
                      variables=sorted(violated),
                      message=(
                          f"Cell '{cell_id}' modified {sorted(violated)} "
                          f"which cell '{prior_cell_id}' (earlier in 
  notebook) reads. "
                          f"This violates Sequential Dataflow Consistency."
                      ),
                  )

          return None

      def _compute_stale(
          self,
          cell_id: str,
          changed: Set[str],
      ) -> List[str]:
          """
          Compute cells that are now stale because their inputs changed.
          Returns cell IDs in document order.
          """
          if not changed:
              return []

          # Get this cell's previous execution sequence (before current)
          prev_record = self.records.get(cell_id)
          my_prev_seq = prev_record.execution_seq if prev_record else 0

          # Direct staleness: cells that ran after us and read changed vars
          directly_stale = set()
          for other_id, record in self.records.items():
              if other_id == cell_id:
                  continue
              # Only consider cells executed after our previous run
              if record.execution_seq > my_prev_seq:
                  if record.reads & changed:
                      directly_stale.add(other_id)

          # Transitive closure
          all_stale = self._propagate_staleness(directly_stale)

          # Return in document order
          return [c for c in self._cell_order if c in all_stale]

      def _propagate_staleness(self, initial: Set[str]) -> Set[str]:
          """Transitive closure: if A is stale and B reads A's outputs, B is
   stale."""
          stale = set(initial)

          changed = True
          while changed:
              changed = False
              for cell_id, record in self.records.items():
                  if cell_id in stale:
                      continue
                  # Does this cell read outputs of any stale cell?
                  for stale_id in list(stale):
                      stale_record = self.records.get(stale_id)
                      if stale_record and (record.reads &
  stale_record.writes):
                          stale.add(cell_id)
                          changed = True
                          break

          return stale

      def reset(self) -> None:
          """Clear all state. Called on kernel restart."""
          self.records.clear()
          self.seq_counter = 0
          self._cell_order = []

  """
  FerretSDCKernel - IPython kernel with Sequential Dataflow Consistency 
  enforcement.

  A simplified kernel focused on SDC. No profiling, no checkpoint magics.
  SDC is always enabled.
  """

  import time
  from typing import Any, Dict, Optional, Set

  from IPython.core.magic import Magics, line_cell_magic, magics_class
  from ipykernel.ipkernel import IPythonKernel
  from ipykernel.kernelapp import IPKernelApp

  from data_ferret.kernel.checkpoint import Checkpoint, Checkpoints
  from data_ferret.kernel.display_helpers import DisplayHelper
  from data_ferret.kernel.models import SDCMetadata, TrackingData
  from data_ferret.kernel.sdc_enforcer import SDCEnforcer
  from data_ferret.kernel.tracking import TrackingDict


  @magics_class
  class FerretSDCKernel(IPythonKernel, Magics):
      """
      IPython kernel with Sequential Dataflow Consistency enforcement.
      
      Features:
      - Variable access tracking (reads/writes per cell)
      - SDC Rule 3 enforcement (no backward mutations)
      - Staleness computation and reporting
      - Cell order management via magic command
      
      SDC is always enabled. No profiling or checkpoint magics.
      """

      implementation = "ferret_sdc"
      implementation_version = "0.1"
      banner = "Ferret SDC Kernel - Sequential Dataflow Consistency"

      def __init__(self, **kwargs):
          super().__init__(**kwargs)
          assert self.shell is not None
          self.shell.register_magics(self)

          # Display helper
          self._display = DisplayHelper()

          # Current cell being executed
          self._cell_id: Optional[str] = None

          # Checkpointing (for pre/post comparison)
          self._checkpoint = Checkpoints()

          # SDC enforcement
          self._sdc = SDCEnforcer(self._checkpoint)

          # Replace user namespace with tracking dict
          self.shell.user_ns = TrackingDict(self.shell.user_ns)

      # 
  =========================================================================
      # Magic Commands
      # 
  =========================================================================

      @line_cell_magic
      def notebook_structure(self, line: str, cell: str = "") -> None:
          """
          Set the notebook cell order for SDC enforcement.
          
          Usage:
              %notebook_structure cell1 cell2 cell3 ...
              
          Or:
              %%notebook_structure
              cell1
              cell2
              cell3
          """
          if cell:
              cell_order = [c.strip() for c in cell.strip().split('\n') if
  c.strip()]
          else:
              cell_order = line.split()

          self._sdc.set_cell_order(cell_order)

      @line_cell_magic
      def sdc_status(self, line: str, cell: str = "") -> None:
          """Display current SDC state."""
          order = self._sdc.cell_order
          records = self._sdc.records

          status_lines = [
              f"Cell order: {order}",
              f"Executed cells: {list(records.keys())}",
              f"Execution counter: {self._sdc.seq_counter}",
          ]

          for cell_id, record in records.items():
              status_lines.append(
                  f"  {cell_id}: reads={sorted(record.reads)}, "
                  f"writes={sorted(record.writes)}, 
  seq={record.execution_seq}"
              )

          self._display.display_icon_and_text("📊", "SDC Status",
  "\n".join(status_lines))

      # 
  =========================================================================
      # Execution
      # 
  =========================================================================

      async def do_execute(
          self,
          code: str,
          silent: bool,
          store_history: bool = True,
          user_expressions: Optional[dict] = None,
          allow_stdin: bool = False,
          *,
          cell_meta: Optional[dict] = None,
          cell_id: Optional[str] = None,
      ) -> dict:
          """
          Execute code with SDC tracking and enforcement.
          """
          # Extract cell context
          self._cell_id = self._extract_cell_id(cell_id, cell_meta)

          # Update cell order if provided in metadata
          if cell_meta and 'cell_order' in cell_meta:
              self._sdc.set_cell_order(cell_meta['cell_order'])

          # Check for notebook_structure magic (parse and remove if present)
          code = self._process_structure_magic(code)

          # Skip SDC for empty code or pure magic
          if not code.strip() or self._is_pure_magic(code):
              return await self._execute_without_sdc(
                  code, silent, store_history, user_expressions,
  allow_stdin, cell_meta
              )

          # Take pre-execution snapshot
          user_ns = self.shell.user_ns
          pre_checkpoint = self._take_pre_snapshot()

          # Reset tracking for this execution
          if isinstance(user_ns, TrackingDict):
              user_ns.reset_tracking()

          # Execute with tracking
          start_time = time.time()

          if isinstance(user_ns, TrackingDict):
              with user_ns.track_execution():
                  result = await super().do_execute(
                      code, silent, store_history, user_expressions,
  allow_stdin,
                      cell_meta=cell_meta, cell_id=self._cell_id,
                  )
              tracking = user_ns.get_tracking_data()
          else:
              result = await super().do_execute(
                  code, silent, store_history, user_expressions,
  allow_stdin,
                  cell_meta=cell_meta, cell_id=self._cell_id,
              )
              tracking = None

          duration = time.time() - start_time

          # Run SDC check if we have tracking data and cell_id
          sdc_result = None
          if tracking and self._cell_id:
              sdc_result = self._sdc.check(
                  cell_id=self._cell_id,
                  pre_checkpoint=pre_checkpoint,
                  post_namespace=dict(self.shell.user_ns),
                  reads=set(tracking.reads_before_writes),
                  writes=set(tracking.writes),
              )

          # Display results
          if result.get("status") != "error":
              self._display_execution_result(duration, tracking, sdc_result)

          # Handle violation
          if sdc_result and sdc_result.violation:
              self._send_violation_error(sdc_result.violation)
              return self._make_error_result(sdc_result.violation)

          return result

      # 
  =========================================================================
      # Helpers
      # 
  =========================================================================

      def _extract_cell_id(
          self, 
          cell_id: Optional[str], 
          cell_meta: Optional[dict]
      ) -> Optional[str]:
          """Extract cell ID from arguments or metadata."""
          if cell_id is not None:
              return cell_id
          if cell_meta is not None:
              return cell_meta.get('cell_id')
          return None

      def _process_structure_magic(self, code: str) -> str:
          """
          Process %notebook_structure magic if present at start of code.
          Removes the magic line and updates cell order.
          Returns remaining code.
          """
          lines = code.split('\n')
          if lines and lines[0].strip().startswith('%notebook_structure'):
              # Extract cell order from magic line
              magic_line = lines[0].strip()
              parts = magic_line.split()[1:]  # Skip the magic name
              if parts:
                  self._sdc.set_cell_order(parts)
              return '\n'.join(lines[1:])
          return code

      def _is_pure_magic(self, code: str) -> bool:
          """Check if code is only magic commands."""
          lines = [l.strip() for l in code.strip().split('\n') if l.strip()]
          return all(l.startswith('%') or l.startswith('!') for l in lines)

      def _take_pre_snapshot(self) -> Checkpoint:
          """Take a snapshot of checkpointable variables before 
  execution."""
          user_ns = self.shell.user_ns
          checkpointable = self._checkpoint.checkpointable_vars(user_ns)
          checkpointable =
  self._checkpoint.checkpointable_values(checkpointable)
          return Checkpoint(f"_pre_{self._cell_id}", checkpointable, {})

      async def _execute_without_sdc(
          self,
          code: str,
          silent: bool,
          store_history: bool,
          user_expressions: Optional[dict],
          allow_stdin: bool,
          cell_meta: Optional[dict],
      ) -> dict:
          """Execute without SDC tracking (for magics, empty code)."""
          return await super().do_execute(
              code, silent, store_history, user_expressions, allow_stdin,
              cell_meta=cell_meta, cell_id=self._cell_id,
          )

      def _display_execution_result(
          self,
          duration: float,
          tracking: Optional[TrackingData],
          sdc_result,
      ) -> None:
          """Display execution timing and SDC metadata."""
          # Build metadata for display
          metadata = SDCMetadata(
              cell_id=self._cell_id or "",
              execution_seq=self._sdc.seq_counter,
              reads=tracking.reads_before_writes if tracking else [],
              writes=tracking.writes if tracking else [],
              changed_variables=sdc_result.changed_variables if sdc_result
  else [],
              stale_cells=sdc_result.stale_cells if sdc_result else [],
              violation=self._violation_to_dict(sdc_result.violation) if
  sdc_result else None,
              cell_order=self._sdc.cell_order,
          )

          # Build display text
          parts = [f"{duration:.2f}s"]
          if tracking:
              if tracking.reads_before_writes:

  parts.append(f"R:{','.join(tracking.reads_before_writes[:3])}")
              if tracking.writes:
                  parts.append(f"W:{','.join(tracking.writes[:3])}")

          if sdc_result and sdc_result.stale_cells:
              parts.append(f"⚠ stale:{','.join(sdc_result.stale_cells)}")

          icon = "✓" if not (sdc_result and sdc_result.violation) else "✗"

          self._display.display_icon_and_text(
              icon,
              " | ".join(parts),
              metadata=metadata.to_display_metadata(),
          )

      def _violation_to_dict(self, violation) -> Optional[Dict[str, Any]]:
          """Convert SDCViolation to dict for JSON serialization."""
          if violation is None:
              return None
          return {
              "mutating_cell": violation.mutating_cell,
              "affected_cell": violation.affected_cell,
              "variables": violation.variables,
              "message": violation.message,
          }

      def _send_violation_error(self, violation) -> None:
          """Send SDC violation as error via iopub."""
          self.send_response(
              self.iopub_socket,
              "error",
              {
                  "ename": "SDCViolation",
                  "evalue": violation.message,
                  "traceback": [
                      "Sequential Dataflow Consistency Violation",
                      "",
                      f"Cell '{violation.mutating_cell}' modified variables 
  that "
                      f"cell '{violation.affected_cell}' (earlier in 
  notebook) reads.",
                      "",
                      f"Affected variables: {violation.variables}",
                      "",
                      "This breaks reproducibility. The earlier cell's 
  behavior "
                      "depends on this cell having run first.",
                  ],
              },
          )

      def _make_error_result(self, violation) -> dict:
          """Create error result dict for SDC violation."""
          return {
              "status": "error",
              "execution_count": self.execution_count,
              "ename": "SDCViolation",
              "evalue": violation.message,
              "traceback": [violation.message],
          }


  # Entry point
  if __name__ == "__main__":
      IPKernelApp.launch_instance(kernel_class=FerretSDCKernel)

  """
  FerretSDCKernelClient - Client that sends cell order with executions.

  Extends BlockingKernelClient to inject cell_id and cell_order
  into execution requests.
  """

  from typing import List, Optional

  from jupyter_client.blocking import BlockingKernelClient


  class FerretSDCKernelClient(BlockingKernelClient):
      """
      Kernel client that sends SDC context (cell_id, cell_order) with 
  executions.
      
      Usage:
          client = FerretSDCKernelClient()
          client.set_cell_order(['cell1', 'cell2', 'cell3'])
          client.execute(code, cell_id='cell2')
      """

      def __init__(self, **kwargs):
          super().__init__(**kwargs)
          self._cell_order: List[str] = []

      def set_cell_order(self, order: List[str]) -> None:
          """Set the notebook cell order for SDC enforcement."""
          self._cell_order = list(order)

      def execute(
          self,
          code: str,
          silent: bool = False,
          store_history: bool = True,
          user_expressions: Optional[dict] = None,
          allow_stdin: Optional[bool] = None,
          stop_on_error: bool = True,
          *,
          cell_id: Optional[str] = None,
          cell_metadata: Optional[dict] = None,
      ) -> str:
          """
          Execute code with SDC context.
          
          Injects cell_id and cell_order into the execution metadata.
          
          Args:
              code: Code to execute
              silent: If True, suppress output
              store_history: If True, store in execution history
              user_expressions: Expressions to evaluate
              allow_stdin: Allow stdin requests
              stop_on_error: Stop on error
              cell_id: ID of the cell being executed
              cell_metadata: Additional cell metadata
              
          Returns:
              Message ID of the execute request
          """
          # Build metadata
          metadata = dict(cell_metadata) if cell_metadata else {}

          if cell_id:
              metadata['cell_id'] = cell_id

          if self._cell_order:
              metadata['cell_order'] = self._cell_order

          # Get the shell channel
          content = {
              'code': code,
              'silent': silent,
              'store_history': store_history,
              'user_expressions': user_expressions or {},
              'allow_stdin': allow_stdin if allow_stdin is not None else
  self.allow_stdin,
              'stop_on_error': stop_on_error,
          }

          # Send with metadata
          msg = self.session.msg('execute_request', content)
          msg['metadata'] = metadata

          self.shell_channel.send(msg)
          return msg['header']['msg_id']

      def execute_with_structure(
          self,
          code: str,
          cell_id: str,
          cell_order: List[str],
          **kwargs,
      ) -> str:
          """
          Convenience method: set order and execute in one call.
          
          Args:
              code: Code to execute
              cell_id: ID of the cell being executed
              cell_order: Full notebook cell order
              **kwargs: Additional arguments to execute()
              
          Returns:
              Message ID
          """
          self.set_cell_order(cell_order)
          return self.execute(code, cell_id=cell_id, **kwargs)

  """Tests for SDC Enforcer."""

  import pytest
  from data_ferret.kernel.checkpoint import Checkpoint, Checkpoints
  from data_ferret.kernel.sdc_enforcer import SDCEnforcer


  class TestSDCEnforcer:

      def setup_method(self):
          self.checkpoints = Checkpoints()
          self.sdc = SDCEnforcer(self.checkpoints)
          self.sdc.set_cell_order(['a', 'b', 'c', 'd'])

      def test_no_violation_forward_dependency(self):
          """Cell B reads what cell A writes - valid."""
          # Cell A writes x
          pre_a = Checkpoint("pre_a", {}, {})
          result_a = self.sdc.check(
              cell_id='a',
              pre_checkpoint=pre_a,
              post_namespace={'x': 1},
              reads=set(),
              writes={'x'},
          )
          assert result_a.violation is None

          # Cell B reads x - valid (forward dependency)
          pre_b = Checkpoint("pre_b", {'x': 1}, {})
          result_b = self.sdc.check(
              cell_id='b',
              pre_checkpoint=pre_b,
              post_namespace={'x': 1, 'y': 2},
              reads={'x'},
              writes={'y'},
          )
          assert result_b.violation is None

      def test_violation_backward_mutation(self):
          """Cell B modifies what cell A reads - violation."""
          # Cell A reads x
          pre_a = Checkpoint("pre_a", {'x': 1}, {})
          self.sdc.check(
              cell_id='a',
              pre_checkpoint=pre_a,
              post_namespace={'x': 1, 'y': 2},
              reads={'x'},
              writes={'y'},
          )

          # Cell B (after A) modifies x - violation!
          pre_b = Checkpoint("pre_b", {'x': 1, 'y': 2}, {})
          result_b = self.sdc.check(
              cell_id='b',
              pre_checkpoint=pre_b,
              post_namespace={'x': 999, 'y': 2},
              reads=set(),
              writes={'x'},
          )

          assert result_b.violation is not None
          assert result_b.violation.mutating_cell == 'b'
          assert result_b.violation.affected_cell == 'a'
          assert 'x' in result_b.violation.variables

      def test_staleness_computation(self):
          """Re-running cell A makes cell B stale if B reads A's output."""
          # First run: A writes x, B reads x
          pre_a = Checkpoint("pre_a", {}, {})
          self.sdc.check('a', pre_a, {'x': 1}, set(), {'x'})

          pre_b = Checkpoint("pre_b", {'x': 1}, {})
          self.sdc.check('b', pre_b, {'x': 1, 'y': 2}, {'x'}, {'y'})

          # Re-run A with different value
          pre_a2 = Checkpoint("pre_a2", {'x': 1, 'y': 2}, {})
          result = self.sdc.check('a', pre_a2, {'x': 100, 'y': 2}, set(),
  {'x'})

          # B should be stale
          assert 'b' in result.stale_cells

      def test_staleness_transitive(self):
          """Staleness propagates transitively."""
          # A writes x, B reads x writes y, C reads y
          self.sdc.check('a', Checkpoint("", {}, {}), {'x': 1}, set(),
  {'x'})
          self.sdc.check('b', Checkpoint("", {'x': 1}, {}), {'x': 1, 'y':
  2}, {'x'}, {'y'})
          self.sdc.check('c', Checkpoint("", {'x': 1, 'y': 2}, {}), {'x': 1,
   'y': 2, 'z': 3}, {'y'}, {'z'})

          # Re-run A
          result = self.sdc.check('a', Checkpoint("", {'x': 1, 'y': 2, 'z':
  3}, {}),
                                  {'x': 100, 'y': 2, 'z': 3}, set(), {'x'})

          # B and C should be stale (B directly, C transitively)
          assert 'b' in result.stale_cells
          assert 'c' in result.stale_cells

      def test_no_staleness_if_value_unchanged(self):
          """Semantic check: no staleness if value didn't actually 
  change."""
          # A writes x=1
          self.sdc.check('a', Checkpoint("", {}, {}), {'x': 1}, set(),
  {'x'})

          # B reads x
          self.sdc.check('b', Checkpoint("", {'x': 1}, {}), {'x': 1, 'y':
  2}, {'x'}, {'y'})

          # Re-run A with same value x=1
          result = self.sdc.check('a', Checkpoint("", {'x': 1, 'y': 2}, {}),
                                  {'x': 1, 'y': 2}, set(), {'x'})

          # B should NOT be stale (x didn't change)
          assert 'b' not in result.stale_cells

      def test_cell_order_update(self):
          """Cell order can be updated, affecting position-based checks."""
          # Initially [a, b, c, d]
          # B reads x
          self.sdc.check('b', Checkpoint("", {'x': 1}, {}), {'x': 1}, {'x'},
   set())

          # A modifies x - currently A is before B, so this is a violation
          result = self.sdc.check('a', Checkpoint("", {'x': 1}, {}), {'x':
  2}, set(), {'x'})
          # Wait - A is before B in order, so A modifying x that B reads is 
  NOT a violation
          # (B depends on A, not the other way around)
          assert result.violation is None

          # Now reorder: [b, a, c, d] - B is now before A
          self.sdc.set_cell_order(['b', 'a', 'c', 'd'])

          # A modifies x - now A is AFTER B, so this IS a violation
          result = self.sdc.check('a', Checkpoint("", {'x': 2}, {}), {'x':
  3}, set(), {'x'})
          assert result.violation is not None
          assert result.violation.affected_cell == 'b'
