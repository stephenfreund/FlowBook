     Plan: Prepare Code for Ferret - Pandas Chained Assignment Rewriter

     Overview

     Add a new server command that uses AST transformation to rewrite 
     problematic pandas chained assignment patterns, preventing 
     ChainedAssignmentError. Users will explicitly call this command to 
     transform their notebooks.

     Approach

     Server Command Architecture - Implement as an explicit command (not 
     automatic kernel transformation) to give users control over when 
     transformations happen.

     LibCST for Transformation - Use LibCST (Concrete Syntax Tree) instead 
     of standard AST to preserve all formatting, whitespace, comments, and 
     code style. This ensures transformed code looks natural and 
     hand-edited.

     What Gets Transformed

     Pattern Detection

     Detect and rewrite: df[subscript].method(..., inplace=True)

     Examples:
     # Before
     df['col'].fillna(0, inplace=True)
     df[mask]['col'].replace(old, new, inplace=True)
     df[df['x'] > 5]['y'].sort_values(inplace=True)

     # After
     df['col'] = df['col'].fillna(0)
     df[mask]['col'] = df[mask]['col'].replace(old, new)
     df[df['x'] > 5]['y'] = df[df['x'] > 5]['y'].sort_values()

     What Doesn't Get Transformed

     - df.fillna(0, inplace=True) - Direct method call (no subscript)
     - df['col'].fillna(0) - No inplace parameter
     - df['col'].fillna(0, inplace=False) - Explicitly False
