  Changes Made

  1. data_ferret/cli/cli.py:
    - Added --force-checkpoints argument (lines 120-124)
    - Added logic to enable force checkpoints after kernel setup (lines
  164-169)
  2. data_ferret/kernel/ferret_kernel.py:
    - Updated checkpoint naming from cell_{cell_id} to pre_{cell_id} (line
  295)
    - Updated checkpoint diff retrieval to match new naming (line 427)

  How It Works

  When you run any command with --force-checkpoints:
  data_ferret execute_all notebook.ipynb --force-checkpoints

  1. The kernel is started/connected
  2. A KernelCommandClient is created
  3. The force_checkpoints(enabled=True) kernel command is sent
  4. The kernel automatically saves a checkpoint before every cell execution
  5. Checkpoints are stored with the naming convention pre_{cell_id} (e.g.,
  pre_abcd, pre_xyzw)

  messages)

  - All cells execute successfully

  ✅ Without --force-checkpoints:
  - No automatic checkpoints
  - Normal execution behavior
  - No performance overhead

  Key Advantages

  - No command modifications: Works with any command that requires a kernel
  - Kernel-level setting: Consistent behavior across all cell executions
  - Clean implementation: All logic contained in CLI layer and kernel
