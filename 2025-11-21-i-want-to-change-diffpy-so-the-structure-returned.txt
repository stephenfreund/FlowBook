  Plan: Support Multiple Differences Per Structure

  Current State

  - Lists/tuples/dicts: Already collect multiple differences as nested dict
  entries ({"[0]": diff, "[1]": diff}) up to max_diffs_per_container
  (default: 1000)
  - Arrays/Series/DataFrames: Use "legacy" methods that return immediately
  on first difference found
  - Type system supports nested diffs via DiffNode = Union[ValueComparison, 
  Dict[str, DiffNode]]

  Proposed Changes

  1. Add New Parameter (max_diffs_per_structure)

  - Add max_diffs_per_structure to Diff.__init__() (default: 5)
  - This provides fine-grained control for structured data (arrays,
  DataFrames, Series)
  - Keep max_diffs_per_container for general containers (lists, dicts)

  2. Update types.py

  - Add MultipleComparisons class to hold a list of ValueComparison objects
  - Update DiffNode type: Union[ValueComparison, MultipleComparisons, 
  Dict[str, DiffNode]]
  - Update format_diff_as_markdown() to handle MultipleComparisons
  - Update serialize_diff_result() to serialize MultipleComparisons

  3. Rewrite Array Comparison (_compare_ndarray)

  - Replace legacy single-error approach
  - Collect up to max_diffs_per_structure element differences
  - Return nested dict structure: {"[0,1]": ValueComparison, "[2,3]": 
  ValueComparison}
  - Add truncation indicator if more diffs exist

  4. Rewrite Series Comparison (_compare_series)

  - Replace legacy single-error approach
  - Collect up to max_diffs_per_structure value differences
  - Return nested dict: {"[index1]": ValueComparison, "[index2]": 
  ValueComparison}
  - Add truncation indicator

  5. Rewrite DataFrame Comparison (_compare_dataframe)

  - Replace legacy single-error approach
  - Collect up to max_diffs_per_structure cell differences across all
  columns
  - Return nested dict: {"['col1'][row1]": ValueComparison, 
  "['col2'][row2]": ValueComparison}
  - Add truncation indicator

  6. Update Markdown Formatter

  - Handle MultipleComparisons type in format_diff_as_markdown()
  - Display multiple diffs at same path with sub-bullets

  Example Output Structure

  # After changes:
  {
    'arr': {
      '[0]': ValueComparison(status='different', message='Array values 
  mismatch at arr[0]: 1 vs 2'),
      '[3]': ValueComparison(status='different', message='Array values 
  mismatch at arr[3]: 4 vs 5'),
      '[7]': ValueComparison(status='different', message='Array values 
  mismatch at arr[7]: 8 vs 9'),
      '_truncated': ValueComparison(status='different', message='Truncated 
  after 5 differences')
    }
  }

  This approach:
  - ✅ Maintains backward compatibility with existing type system
  - ✅ Reuses the nested dict pattern already used for lists/dicts
  - ✅ Adds fine-grained control via new parameter
  - ✅ Provides up to N differences per structure (configurable)

⏺ Performance Fix Summary

  Good catch! I had removed the fast vectorized check that was critical for
  performance. Here's what I fixed:

  The Problem

  My initial implementation always iterated element-by-element through
  arrays/Series, even when they were equal. This was ~100x slower than the
  vectorized numpy operations.

  The Fix

  I added back the fast path for all structure types:

  1. Arrays (diff.py:715-726):
    - Use np.allclose() for float arrays (vectorized, fast)
    - Use np.array_equal() for other arrays (vectorized, fast)
    - Return None immediately if equal
    - Only iterate element-by-element if different
  2. Series (diff.py:840-856):
    - Use Series.equals() for non-float types
    - Use np.allclose() on non-NaN values for float types
    - Return None immediately if equal
    - Only iterate element-by-element if different
  3. DataFrames:
    - Benefit from Series fast path since we compare column-by-column

  Performance Results

  For 10k element arrays:
  - Equal arrays: ~0.10 ms (fast vectorized check)
  - Different arrays: ~0.34 ms (finds first 5 diffs then exits)

  The fast path makes equal comparisons 100x+ faster while maintaining the
  ability to collect multiple differences when arrays differ. All 219 tests
  still pass!
