             for cell in cells:

     store_history=False)

     store_history=False)

     Key Design Decisions

     1. Use checkpoints (not direct namespace comparison): Saves pre and 
     post checkpoints, then compares them. This ensures consistency and
     leverages existing checkpoint infrastructure.
     2. Leq semantics: Using use_leq=True in Diff allows:
       - Extra keys in post (new variables created by cell)
       - Extra DataFrame columns in post
       - Only flags actual modifications to read-before-write variables
     3. Cleanup checkpoints: Delete pre/post checkpoints after each cell to 
     avoid memory bloat during long notebooks.
     4. Error reporting: Dual reporting via:
       - Cell output (visible in JupyterLab)
       - Metadata (accessible programmatically)
