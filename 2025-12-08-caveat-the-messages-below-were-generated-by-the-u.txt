  Bug Summary

  Root Cause: In _execute_with_profiling() (ferret_kernel.py:484),
  type_models() was called AFTER start_column_tracking(). The type_models()
  function calls get_type_model() which accesses every DataFrame column via
  obj[col].dtype (extended_types.py:233). Since column tracking was active,
  this triggered the patched __getitem__ and recorded reads for ALL columns
  of ALL DataFrames before any user code ran.

  Result: The column_reads_before_writes metadata incorrectly showed all
  columns of all DataFrames (train, test, df) as being "read" even when the
  cell only accessed df['date'].

  Fix Applied

  Restructured _execute_with_profiling() to ensure type introspection
  happens OUTSIDE the column tracking window:

  1. Capture pre_types BEFORE starting column tracking (line 498-502)
  2. Start column tracking (line 504-506)
  3. Execute user code
  4. Capture tracking data while tracking is still active (line 527-528,
  540-541)
  5. Stop column tracking in finally block (line 542-545)
  6. Capture post_types AFTER stopping column tracking (line 547-551)

  This ensures the column access tracking only captures actual user code
  access patterns, not internal type introspection operations.
