  Summary

  The new module provides:

  Core Features

  - AST-based analysis using Python's ast module to parse code cells
  - Flow-insensitive tracking of global variable accesses
  - Function call detection including functions passed as arguments to library functions
  - Scope-aware analysis that distinguishes local vs global variables

  Main Components

  1. GlobalAccessAnalyzer - AST visitor that tracks:
    - Global variables read
    - Global variables written
    - Functions called (including those passed as arguments)
    - Proper scoping for functions, classes, lambdas, and loops
  2. CellDependencies - Data class storing per-cell dependency information
  3. analyze_notebook() - Analyzes all code cells in a notebook and returns a map from cell ID to dependencies
  4. get_dependency_graph() - Simplified interface returning cell ID → list of globals accessed

  Example Usage

      ]

    - For writes: add to written_at_module_level and globals_written

  - If statements: Variables written inside conditionals are "maybe written"
    - Don't add conditional writes to written_at_module_level
    - Be conservative: treat as not-definitely-written
  - Loops: Similar to conditionals - conservative treatment
  - Try/except: Writes in try blocks might not execute

  4. Function Definitions

  - Function bodies are analyzed separately (already done)
  - The function definition itself is a write at module level
  - Track when function is defined vs when it's called
