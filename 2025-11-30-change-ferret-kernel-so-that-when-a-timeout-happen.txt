     Cell Timeout Error Handling Plan

     Problem Statement

     When a cell execution times out (after 30 minutes by default), the 
     CellTimeoutHandler raises a KeyboardInterrupt to stop execution. 
     However, this exception is not caught in the do_execute() method, 
     resulting in:
     - No error message sent to the client via iopub socket
     - Silent failure from the perspective of both KernelHelper and Jupyter 
     UI
     - The finally block executes cleanup, but no error result is returned

     Root Cause Analysis

     File: data_ferret/kernel/ferret_kernel.py

     Current Flow:
     1. Line 304: timeout_handler.start() arms the timeout watchdog
     2. Line 305: normal_exit = False
     3. Lines 307-325: Execution happens in try block
     4. On timeout: CellTimeoutHandler._on_timeout() calls 
     _thread.interrupt_main() which raises KeyboardInterrupt
     5. KeyboardInterrupt is NOT caught, so execution jumps to finally block
     6. Lines 327-330: Cleanup occurs but NO error result is returned
     7. The KeyboardInterrupt propagates up, but the IPython kernel swallows
      it without sending proper error to client

     Existing Error Pattern (lines 370-375):
     When Scalene detects an error, it properly sends via iopub:
     if result.get('status') == 'error':
         self.send_response(self.iopub_socket, 'error', {
             'ename': result.get('ename', 'UnknownError'),
             'evalue': result.get('evalue', ''),
             'traceback': result.get('traceback', [])
         })

     Solution

     Add exception handling for KeyboardInterrupt in the do_execute() method
      to:
     1. Catch the timeout interrupt
     2. Send a proper error message via iopub socket
     3. Return a proper error result dict

     Implementation Details

     File to Modify

     Changes to do_execute() method (lines 276-330)

     Add exception handling after the try block:

     async def do_execute(
         self,
         code: str,
         silent: bool,
         store_history: bool = True,
         user_expressions: Optional[dict] = None,
         allow_stdin: bool = False,
         *,
         cell_meta: Optional[dict] = None,
         cell_id: Optional[str] = None,
     ) -> dict:
         """Execute code with optional profiling and checkpointing."""
         self._extract_cell_id(cell_id, cell_meta)
         code, timeout = self._parse_timeout_from_code(code)
         self.display_cell_id()

         force_checkpoints = self._force_checkpoints

         if force_checkpoints:
             self.checkpoint(f"save pre_{self._cell_id}")

         timeout_handler = CellTimeoutHandler(
             timeout=timeout,
             post_kb_grace=self._post_kb_grace,
             kill_timeout=self._kill_timeout,
             verbose=self._verbose,
             max_passes=self._max_passes,
         )
         timeout_handler.start()
         normal_exit = False

         try:
             start_time = time.time()
             result, profile_contents, pre_types, post_types = await 
     self._execute_with_profiling(
                 code, silent, store_history, user_expressions, allow_stdin,
      cell_meta
             )
             normal_exit = True
             end_time = time.time()

             # Only display execution result if not an error
             if result.get('status') != 'error':
                 self._display_execution_result(
                     end_time - start_time, profile_contents, pre_types, 
     post_types, code
                 )

             if force_checkpoints:
                 self._show_checkpoint_diff()

             return result

         except KeyboardInterrupt:
             # Cell execution timed out
             timeout_msg = f"Cell execution timed out after {timeout} 
     seconds"

             # Send error to client via iopub socket
             self.send_response(self.iopub_socket, 'error', {
                 'ename': 'TimeoutError',
                 'evalue': timeout_msg,
                 'traceback': [timeout_msg]
             })

             # Return error result
             return {
                 'status': 'error',
                 'execution_count': self.execution_count,
                 'ename': 'TimeoutError',
                 'evalue': timeout_msg,
                 'traceback': [timeout_msg]
             }

         finally:
             timeout_handler.cancel()
             if not normal_exit:
                 await timeout_handler.cleanup_on_error()

     Key Design Decisions

     1. Exception Type: Catch KeyboardInterrupt specifically, which is what 
     _thread.interrupt_main() raises

     execution timed out after {timeout} seconds"
     4. ename: Use 'TimeoutError' as the exception name for semantic clarity
     5. Traceback: Keep it simple with just the timeout message (no need for
      full traceback since this is a timeout, not a code exception)
     6. Return Value: Match the structure used by Scalene error returns 
     (lines 82-88 in scalene_runner.py)

     Problem Statement

     The execute_all command currently lacks robust error handling compared 
     to the profile command. Specifically:

     1. No error detection: Doesn't check if result["status"] == "error" 
     after cell execution
     2. No error stopping: Continues executing all cells even if one fails
     3. No status tracking: Always returns "status": "success" regardless of
      cell failures
     4. No exception handling: Lacks try-except blocks to catch Python 
     exceptions
     5. No error reporting: Doesn't print error messages to console for 
     debugging
     6. Short timeout: Uses default 30-second timeout instead of 30-minute 
     kernel timeout

     execution.
