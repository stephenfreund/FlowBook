⏺ Now let me think through case #7 more thoroughly. The question is: what
  AST patterns result in calling a function that we can't statically
  determine?

  Patterns where we CAN track the called function:
  - func() - direct call to ast.Name
  - obj.method() - call to ast.Attribute (we track method name)

  Patterns where we CANNOT track the called function:
  - funcs[0]() - ast.Subscript (list/array indexing)
  - func_dict['key']() - ast.Subscript (dict key access)
  - df.iloc[0]() - ast.Subscript (DataFrame/Series indexing)
  - getattr(obj, name)() - ast.Call as the function (covered by case 2)
  - (lambda: x)() - ast.Lambda as the function (inline lambda)
  - (func1 if cond else func2)() - ast.IfExp (conditional expression)
  - (funcs.pop())() - result of method call used as function

  The key insight: any call where node.func is NOT ast.Name or ast.Attribute
   is an indirect call. This is a simple, comprehensive rule.
